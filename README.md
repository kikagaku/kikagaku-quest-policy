# キカガク Quest 受験ポリシー

株式会社キカガクが提供する eラーニング学習プラットフォーム[キカガク](https://www.kikagaku.ai)内にあるサービス、キカガク Quest の受験ポリシーです。

## キカガク Quest とは

キカガク Quest とは弊社独自のテストを独自のスキル軸毎に受講していただき、エンジニアとして必要なスキルを可視化するサービスです。スキル軸は以下の 5 軸で定義されています。

テストは 2 種類あり、スキル軸に関する知識を問う**知識力テスト**とエンジニアとしての実装力・スキルを問う**実践力テスト**があります。知識力テストは選択問題形式でそれぞれのスキル軸に対してレベルを 3 段階用意しています。実践力テストはテストがあるスキル軸と無いスキル軸があります。実践力テストはテストによって問題形式が若干異なりますが、基本プログラミングが必要なテスト内容です。

※ スキル軸：機械学習（ML/DL）、データサイエンス（DS）、データエンジニアリング（DE）、コンピュータサイエンス（CS）、ソフトウェアエンジニアリング（SE）

| スキル軸 | レベル      |
| -------- | ----------- |
| ML/DL    | Lv.1 ~ Lv.3 |
| DS       | Lv.1 ~ Lv.3 |
| DE       | Lv.1 ~ Lv.3 |
| CS       | Lv.1 ~ Lv.3 |
| SE       | Lv.1 ~ Lv.3 |

## その他試験ポリシー

### 受験パターン

| パターン         | 説明                                                                                             | 補足                                                                                                         |
| ---------------- | ------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------ |
| スキル軸単体受験（共通） | 知識力・実践力ともに好きなレベルからテストを受験できる                                           | 上位レベルをクリアしても下位レベルには影響しない（上位レベルのテストを合格しても下位レベル合格にはならない） |
| スキル軸一括受験<br>（知識力のみ） | 知識力はスキル軸単位で一括受験することができ、今の自分がどのレベルになるかを一度のテストで測れる | 各スキル軸で 1 つもレベルが認定されていない場合のみ受験可能                                                  |

### 認定試験有効期限

一度合格したテスト（スキル軸）には有効期限はございません。  

※ テスト内容が大幅に改修・変更される場合にはこの限りではありません。

### 受験回数制限

| テスト       | 受験回数制限          |
| ------------ | --------------------- |
| 知識力テスト | なし                  |
| 実践力テスト | あり（最大 3 回まで） |

### 再受験

不正行為防止のために再受験にインターバルを設定しています。基本的にどのテストも一度不合格になった際にはインターバルが必要ですが、イレギュラーで受験回数制限 1 回のテストも存在します。

| 受験回     | インターバル       |
| ---------- | ------------------ |
| 1 回目     | -                  |
| 2 回目     | 1 回目から 7 日後  |
| 3 回目以降 | 2 回目から 14 日後 |

### 制限時間

制限時間が設定されていない場合には Web 検索等が可能になるため、制限時間を設定しています。

| テスト                     | 時間                                                              |
| -------------------------- | ----------------------------------------------------------------- |
| 知識力テスト               | 1 問 1 分                                                         |
| 実践力テスト（ML/DL 以外） | 制限時間制限時間までに 1 回だけ提出可能<br>制限時間は問題毎に設定 |
| 実践力テスト（ML/DL）      | 制限時間までに問題毎に 3 回まで提出可能<br>制限時間は問題毎に設定 |

※ ML/DL はシステム側で持っている正解データを基準に採点が行われるため、ユーザーがローカル（自身の作業環境）で合格基準に達していても採点は不合格になってしまうリスクが大きい（本来合格にしたい人が不合格になってしまう）ことが想定されるため 1 回ではなく、期限内に数回提出できるようにしています。ただし、無制限に提出できるようにすると毎回の FB で与えられる精度を基準にチューニングされてしまうリスクがあるため、3 回の上限を設けます。

### テスト中断

| テスト                               | 中断可否                                                                                                 |
| ------------------------------------ | -------------------------------------------------------------------------------------------------------- |
| 知識力テスト                         | **不可**<br>  ※受験途中にウィンドウを閉じたり、別画面に遷移しようとした場合にはモーダルで不合格になる旨を伝える |
| 実践力テスト                         | **可能**<br>  中断ボタンを用意する                                                                             |
| 実践力テスト（イレギュラーパターン） | 全問一括出題画面<br>各パートのテスト開始画面                                                             |

### 解答形式

| テスト       | 形式                                                                                                                                                                                                                            |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 知識力テスト | 単一選択のみ                                                                                                                                                                                                                    |
| 実践力テスト | ・選択型（単一・複数）<br>・数値記述型（整数・少数）<br>・学習済みモデル提出型（テストシステム側で推論処理 → 合否判定）<br>・ファイルアップロード型（分析レポート提出 → 講師による手動採点）<br>・資格提出型（主にクラウド系統） |
